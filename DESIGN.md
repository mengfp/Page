# Page — 设计文档（草案）

> 保密笔记本：本地、口令加密、结构简单。  
> **本文与实现对齐，讨论、改需求时同步改本文 + 代码。**

---

## 1. 产品目标

- 单机笔记，数据进 `.page` 文件（JSON + age）。
- **简单可靠**优先，避免过度设计（无 UUID、无条目间引用等，除非以后明确要）。

---

## 2. 数据模型


| 实体        | 说明                                                                |
| --------- | ----------------------------------------------------------------- |
| **Store** | 内存中的一组 `Entry`，序列化为 **JSON 数组** 再加密。                              |
| **Entry** | `title`, `tags[]`, `content`, `modified`（ISO 8601）。**title 可重复**。 |


磁盘上的文件：`path` + 口令；内存：`dirty` 表示相对磁盘是否有未保存改动。

---

## 3. 初始状态（启动完成）


| 项目   | 状态                                                               |
| ---- | ---------------------------------------------------------------- |
| 文档   | 新文档：`path = null`，`dirty = false`，`entries = []`。                |
| 左侧列表 | 空；**无选中**。                                                       |
| 右侧   | **空白草稿**：新 `Entry` 实例，`pending_add = true`，**Modified 不显示**（空白）。 |
| 标题栏  | `Page - Untitled`（无 `*`）。                                        |


---

## 4. 概念：草稿 vs 已入库


| 概念      | 含义                                                                                    |
| ------- | ------------------------------------------------------------------------------------- |
| **草稿**  | 右侧正在编的 `Entry`，**尚未** `add_entry` 进 `store.entries`。`pending_add = true`，Modified 空白。 |
| **已入库** | 已在 `store.entries` 里；从列表点开会 `pending_add = false`，Modified 显示时间。                      |


**New Entry / reset_to_new_draft** 永远是 **新的 `Entry()` 实例**，避免和已入库对象混用导致「覆盖」感。

---

## 5. 动作一览（当前行为）

### 5.1 File → New

1. **确认放弃**（见 §6）；Cancel → 结束。
2. `App.new()`；列表刷新为空；**无选中**；右侧 **空白草稿**。
3. 状态栏：`New file created.`

### 5.2 File → Open

1. **确认放弃**；Cancel → 结束。
2. 选文件 → 口令（不允许空）→ 解密载入。
3. 失败 → 报错，界面不变。
4. 成功：**无选中**；右侧 **空白草稿**（不自动打开某一条）。

### 5.3 File → Save

- 尚无 path → 等价「首次保存」：路径 + 新口令 → `save_as` → `dirty = false`。
- 已有 path → `save()` → `dirty = false`。

### 5.4 File → Save As

路径 + 新口令 → `save_as` → 更新当前 path / 口令 → `dirty = false`。

### 5.5 关闭窗口 / Quit

**确认放弃**（§6）；Discard 才关闭。

---

### 5.6 Entry → New Entry（含左侧「New Entry」按钮）

**点 New Entry 时，右侧若是：**

| 右侧状态 | 是否弹窗 | 行为 |
|----------|----------|------|
| **1. 空白草稿**（`pending_add` 且标题/标签/正文全空） | 否 | **无操作**（已是同一状态，不重复 reset）。 |
| **2. 已入库且无改动**（列表选中某条，表单与 `_entry` 一致） | 否 | 列表 **取消选中**；右侧 **空白草稿**（新 `Entry()`），方便接着写新条。 |
| **非空草稿** 或 **已入库但有未 Apply 修改** | 是 | 确认放弃后再同上（无选中 + 空白草稿）。 |

实现：`uncommitted_input()` 仅在「非空草稿」或「表单与已入库不一致」时为真；1、2 均为假，故不弹窗。

### 5.7 左侧列表：选中某条

- 若存在 **非空草稿** 或 **已入库未 Apply**（§6）→ 先问是否放弃再切换；**No** → 保持原选中行、右侧不变。
- 否则：右侧加载该条，`pending_add = false`，Modified 显示。

### 5.8 Entry → Delete Entry

- 列表无选中 → **无操作**。
- 有选中 → 确认 → 从 store 删除 → 刷新 → **无选中** → 右侧 **空白草稿** → `dirty = true`。

---

### 5.9 右侧 Apply

1. 表单写入当前 `_entry`，`touch()`，发 `entry_changed(entry, true)`。

**MainWindow：**

- `**entry` 不在 `store.entries`**（草稿首次 Apply）  
→ `add_entry` → 刷新列表 → **无选中** → `**reset_to_new_draft()`**（下一条必须是新对象）。
- **已在 store**（从列表打开后编辑）  
→ `update_entry` → 刷新 → **仍选中该条**。

### 5.10 右侧 Cancel

- **草稿** → 再 `**reset_to_new_draft()`**（新空白草稿）。
- **已入库条目** → 表单恢复为该条已 Apply 的内容（不写盘）。

---

## 6. 何时提示「放弃？」

在 **File → New / Open**、**关窗口** 前：

- `dirty == true`，或  
- **非空草稿**，或  
- **已打开条目且右侧与已 Apply 内容不一致**（未 Apply 的编辑）。

空草稿、未改已打开条目、且未 dirty → 不提示。  

（`uncommitted_input()` 已包含上述后两项逻辑，供 MainWindow 使用。）
