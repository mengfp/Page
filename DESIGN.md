# Page — 设计文档（草案）

> 保密笔记本：本地、口令加密、结构简单。  
> **本文与实现对齐，讨论、改需求时同步改本文 + 代码。**  
> 只记**非显然**约定；能从上文直接推出的不重复写（修订表从简，以代码为准）。

---

## 1. 产品目标

- 单机笔记，数据进 `.page` 文件（JSON + age）。
- **简单可靠**优先，避免过度设计（无 UUID、无条目间引用等，除非以后明确要）。
- **界面文案 ≠ 代码命名**：给用户看的标签（如 Date）不必与字段名（如 `modified`）同一词；代码以清晰、稳定为主，界面以好懂为主。

---

## 2. 数据模型


| 实体        | 说明                                                                |
| --------- | ----------------------------------------------------------------- |
| **Store** | 内存中的一组 `Entry`，序列化为 **JSON 数组** 再加密。                              |
| **Entry** | `title`, `tags[]`, `content`, `modified`。标签为 **小 chip**；**Add** 打开输入（可补全）；**右键 chip → Delete**。 |


磁盘上的文件：`path` + 口令；内存：`dirty` 表示相对磁盘是否有未保存改动。

---

## 3. 初始状态（启动完成）


| 项目   | 状态                                                               |
| ---- | ---------------------------------------------------------------- |
| 文档   | 新文档：`path = null`，`dirty = false`，`entries = []`。                |
| 左侧列表 | 空表（Title / Date / Tags）；**无选中**。                                 |
| 右侧   | **空白草稿**：新 `Entry` 实例，`pending_add = true`，**Date 栏不显示**（空白）。 |
| 标题栏  | `Page - Untitled`（无 `*`）。                                        |


---

## 4. 概念：草稿 vs 已入库


| 概念      | 含义                                                                                    |
| ------- | ------------------------------------------------------------------------------------- |
| **草稿**  | 右侧正在编的 `Entry`，**尚未** `add_entry` 进 `store.entries`。`pending_add = true`，Date 栏空白。**Apply 才进列表。** |
| **已入库** | 已在 `store.entries` 里；从列表点开会 `pending_add = false`，Date 栏显示 `modified` 时间。                      |


**New draft**（右侧按钮 **New**）= 换一张空白草稿（新 `Entry()`），**不是**往列表里插一条。列表只在 **Apply** 时增加一行。

---

## 5. 动作一览（当前行为）

### 5.1 File → New

1. **确认放弃**（见 §6）；Cancel → 结束。
2. `App.new()`；列表刷新为空；**无选中**；右侧 **空白草稿**。
3. 状态栏：`New file created.`

### 5.2 File → Open

1. **确认放弃**；Cancel → 结束。
2. 选文件 → 口令（不允许空）→ 解密载入。
3. 失败 → 报错，界面不变。
4. 成功：**无选中**；右侧 **空白草稿**（不自动打开某一条）。

### 5.3 File → Save

- **先 Apply（进内存）再写盘**；无单独「只 Flush」。
- 尚无 path → 路径 + 新口令 → `save_as` → `dirty = false`。
- 已有 path → `save()` → `dirty = false`。

### 5.4 File → Save As

同 Save：先 Apply 再选路径 + 口令 → `save_as`。

### 5.5 关闭 / New / Open（有未保存时）

**Save | Discard | Cancel**。Save 等同 File → Save（Apply + 写盘）；Discard 放弃；Cancel 不打断。

### 5.6 Help → About Page

- 显示应用名、版本号（见 `version.py`）、一句说明（本地加密、数据在本机）。

---

### 5.7 New draft（右侧 **New**）

理念：**新草稿**，不是「在容器里插入新 entry」；入库仅通过 **Apply**。

| 右侧状态 | 是否弹窗 | 行为 |
|----------|----------|------|
| **1. 空白草稿**（`pending_add` 且标题/标签/正文全空） | 否 | **无操作**。 |
| **2. 已入库且无改动**（列表选中某条，表单与 `_entry` 一致） | 否 | 列表 **取消选中**；右侧 **空白草稿**（新 `Entry()`）。 |
| **非空草稿** 或 **已入库但有未 Apply 修改** | 是 | 确认后再：无选中 + 空白草稿。 |

### 5.8 左侧列表：选中某条

- 若存在 **非空草稿** 或 **已入库未 Apply**（§6）→ 先问是否放弃再切换；**Cancel** → 保持原选中行、右侧不变。
- 否则：右侧加载该条，`pending_add = false`，Date 栏显示。

### 5.9 Delete（**列表右键 → Delete**；快捷键 **Delete** 删当前选中条）

- 右键点在行上 → **Delete** → 确认 → 删该条。
- 快捷键 **Delete**：列表有选中时同逻辑；无选中 → **无操作**。

---

### 5.10 右侧 Apply

1. **标题必填**（strip 后非空）；否则提示，不写入。
2. 否则表单写入 `_entry`，`touch()`，发 `entry_changed(entry, true)`（与上次已 Apply 内容无差异时不 `touch`、不 emit，见实现）。

**MainWindow：**

- **entry 不在 `store.entries`**（草稿首次 Apply）  
→ `add_entry` → 刷新列表 → **无选中** → **空白草稿**（下一条必须是新对象）。
- **已在 store**（从列表打开后编辑）  
→ `update_entry` → 刷新 → **仍选中该条**。

### 5.11 右侧 Cancel

- **草稿** → 再 **空白草稿**。
- **已入库条目** → 表单恢复为该条已 Apply 的内容（不写盘）。

---

## 6. 何时提示 Save / Discard？

在 **File → New / Open**、**关窗口** 前，若 **dirty** 或 **未 Apply 的编辑**（含非空草稿）→ 三键对话框；否则不提示。
